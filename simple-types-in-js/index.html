Simple Types in JS

posted by Beka on 23 Dec 2014

In this post, I'm going to show you how to implement a type checker using JavaScript. JS is not the most ideal language for this, unfortunately, because it lacks a good type system of its own, and makes a bunch of stuff really ugly and noisy, but in a way, that's ideal. It'll give you a sense of what you'd have to do in other languages with similarly impoverished type systems. The way I'm going to implement it should be relatively easy to translate into idiomatic code in various.

I'm assuming you know a little bit about type theory, such as having looked at the Pfenning videos that I posted in this little guide. If not, I highly recommend you brush up, otherwise some of this might be confusing.

Our first task is to figure out just what the type theory is that we want to implement. To keep things simple, we'll implement only four types (and add some more later). We'll have pair types and function types, which will be the interesting cases, and we'll also have two types Foo and Bar which won't do anything except give us two base types to work with. We won't be able to do much programming with just these, but we'll still be able to see how a type checker works.

So let's define the type theory. First we'll define a judgment A type which will serve as an internal way of represent when something is or isn't a well-formed type:

A type
